# Air Landing Page

A responsive landing page for Air, a strategic creative agency, created to present the agency's services, expertise, process, testimonials, and contact details.

## Project Description

The Air Landing Page is a single-page website designed to introduce visitors to a modern creative agency. It includes sections for the agency overview, areas of expertise, working process, client testimonials, a call-to-action block, and contact information.

The page is built according to the provided Figma design and focuses on a clean responsive layout, smooth interactions, and a user-friendly experience across mobile, tablet, and desktop screens.

[Visit the Website](https://opalahecha.github.io/landing_dia/) | [View Figma Design](https://www.figma.com/design/7qwsWggv9BAxMi2VPhBuPr/Air--formerly-Dia--?node-id=22904-2182)

## Technical Requirements

To run this project, you will need:

- Node.js `22.18.0` or newer: A JavaScript runtime used to run the project tooling.
- NPM `11.13.0` or newer: The Node.js package manager used for installing dependencies and running scripts.
- Git: A version control system used to clone and manage the project.

## Installation and Setup

To install the project and run it locally, follow these steps:

1. Clone the repository:

```bash
git clone https://github.com/opalahecha/landing_dia.git
```

2. Navigate to the project directory:

```bash
cd landing_dia
```

3. Install dependencies:

```bash
npm install
```

4. Start the local development server:

```bash
npm start
```

5. Build the project for production:

```bash
npm run build
```

## Usage

After starting the project, it will be available in the browser at the local development URL shown in the terminal, usually `http://localhost:8080`.

You can use this URL to preview the landing page, test responsive layouts, and check interactive elements. Any changes made to the source files will be reflected during development.

## Features

- Responsive design for mobile, tablet, desktop, and large screens.
- Header navigation with links to page sections.
- Toggleable mobile menu.
- Hero section with an image slider.
- Expertise section with reusable cards.
- Process section with interactive cards.
- Testimonials section with responsive card layout.
- Vision, Passion, Results call-to-action section.
- Contact form with native validation.
- Form reset and smooth scroll after submit.
- Google Maps link for the office address.
- Smooth scrolling across the page.

## Technologies Used

This project was built using the following technologies:

- HTML5: For creating semantic and accessible page markup.
- CSS3: For styling the page and creating responsive layouts.
- Sass: A CSS preprocessor used for variables, mixins, nesting, and modular styles.
- JavaScript: For slider functionality, mobile menu behavior, form handling, and smooth interactions.
- BEM methodology: For writing structured and maintainable class names.
- Flexbox: For flexible layout alignment.
- CSS Grid: For building responsive page sections.
- Parcel: A web application bundler used by the project tooling.
- Node.js: For running the development environment.
- NPM: For managing project dependencies and scripts.
- Git: For version control.
- GitHub: For repository hosting and deployment through GitHub Pages.
